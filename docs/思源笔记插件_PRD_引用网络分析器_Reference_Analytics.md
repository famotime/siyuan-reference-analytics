## 一、项目名称

脉络镜 Context lens

## 二、项目背景

在知识管理系统中，图谱功能常常被当作一个“看起来很强”的展示能力，但对大多数专业用户来说，单纯可视化节点和连线并不能真正提升决策效率。

用户真正关心的问题不是图谱是否炫酷，而是：

- 哪些文档块是当前知识网络中的核心节点
- 哪些主题已经形成稳定知识簇
- 哪些文档正在孤立、断裂或沉没
- 哪些文档正在变得更重要，哪些文档正在失去连接价值

因此，脉络镜的核心价值不应停留在“画图”，而应转向“分析、筛选、解释与行动建议”。

## 三、项目目标

### 目标

- 帮助用户识别知识网络中的核心文档块与高价值节点
- 帮助用户发现主题社区、知识簇和连接结构
- 帮助用户识别孤立文档、断裂区域和沉没资料
- 帮助用户从时间维度理解知识网络的变化趋势
- 为整理、索引建设、专题沉淀、知识修复提供明确入口

### 非目标

- 不在第一阶段实现复杂图数据库能力
- 不在第一阶段做大型交互式 3D 图谱
- 不在第一阶段引入不可解释的黑箱分析模型
- 不在第一阶段覆盖所有高级网络科学指标
- 不在第一阶段提供任意内容块级别的全量排行与社区分析

## 四、目标用户

### 核心用户

- 重视知识连接和主题沉淀的思源用户
- 做研究、写作、资料整理、项目知识管理的中重度用户
- 需要从大量引用关系中识别结构规律的人

### 次级用户

- 使用双链和文档引用较多的高级用户
- 希望优化知识图谱但不满足于原始可视化的用户
- 想通过数据辅助整理知识库结构的人

## 五、用户痛点

### 痛点 1：不知道哪些文档最核心

用户能看到大量引用关系，但无法判断哪些文档块真正位于知识网络中心，哪些文档值得升级为索引页、主题页或关键资产。

### 痛点 2：不知道哪些主题形成知识簇

图谱中虽然有很多连接，但难以看出哪些内容已经聚集成主题社区，哪些主题仍然松散。

### 痛点 3：不知道哪些知识断裂或沉没

很多文档写完后没有后续连接，逐渐变成孤立节点或长期沉没资料，用户缺乏发现和修复入口。

### 痛点 4：缺少时间维度

用户很难知道某个主题是正在升温，还是已经长期失活；也难判断某个核心文档是持续重要，还是短期热点。

## 六、核心使用场景

### 场景 1：识别知识核心节点

用户希望找出被其他文档引用最频繁、连接最强的文档块，用于建设主题页、导航页和索引页。

### 场景 2：发现主题社区

用户希望知道自己的知识库中已经自然形成了哪些知识簇，以便整理成专题结构。

### 场景 3：发现断裂内容

用户希望找出孤立文档块、低连接文档和长期未被引用的资料，补充链接或归档。

### 场景 4：观察知识网络变化

用户希望按近 7 天、30 天、90 天等时间窗口查看哪些主题在活跃、哪些主题在沉没。

### 场景 5：分析关系传播路径

用户希望理解某个核心文档如何连接多个主题，或者某个主题内部是通过哪些关键文档串联起来的。

## 七、产品定位

脉络镜不是“另一个图谱视图”，而是面向知识结构优化的分析插件。

它的输出应包括：

- 分析结果
- 筛选视图
- 风险识别
- 整理建议
- 原始文档跳转入口
- 原始引用证据查看入口

## 八、核心功能设计

## 1. 文档块引用热度排行 Document Reference Ranking

### 功能目标

识别知识网络中最常被引用、最具中心性的文档块。

### 核心能力

- 统计文档块被其他文档块引用的次数
- 支持按时间范围筛选，例如近 7 天、30 天、全部时间
- 支持按文档、笔记本、标签、主题过滤
- 展示文档标题、引用文档数、引用次数、最近被引用时间
- 支持查看该文档对应的原始引用证据
- 支持跳转原始文档

### 用户价值

帮助用户识别高价值文档资产，便于升级为索引页、主题页或主干节点。

## 2. 主题社区发现 Topic Community Detection

### 功能目标

从文档块之间的引用关系中发现自然形成的主题簇或知识社区。

### 核心能力

- 基于文档块之间的引用关系做社区聚合
- 展示主题社区列表及社区规模
- 标识社区内部高连接文档
- 支持进入社区查看相关文档分布
- 支持按标签、属性或文档范围辅助解释社区语义

### 用户价值

帮助用户理解知识库中已经成形的主题结构，而不是依赖手工观察图谱。

## 3. 孤立文档块识别 Orphan Document Detection

### 功能目标

识别没有文档级引用关系或连接极弱的文档内容。

### 核心能力

- 统计孤立文档块数量
- 展示孤立文档块清单
- 支持按创建时间、更新时间排序
- 支持筛选长期未被引用内容
- 支持查看该文档是否仅存在零散块级引用证据
- 支持跳转原始文档进行整理

### 用户价值

帮助用户发现知识断裂区，减少内容沉没。

## 4. 时间趋势分析 Trend Analysis

### 功能目标

从时间维度理解文档网络变化，识别热点主题与沉没主题。

### 核心能力

- 统计文档块引用变化趋势
- 展示主题活跃度变化
- 展示新增连接与断裂连接趋势
- 支持按近 7 天、30 天、90 天切换
- 支持识别持续升温主题与长期低活跃主题

### 用户价值

帮助用户判断哪些主题值得继续投入，哪些主题需要复盘或归档。

## 5. 关系传播路径 Path Analysis

### 功能目标

帮助用户理解某个核心文档如何连接多个主题，或两个主题之间通过哪些关键文档建立联系。

### 核心能力

- 查看指定文档到其他核心文档的连接路径
- 查看主题内部关键桥接文档
- 标识高传播价值节点
- 支持限定深度与路径范围
- 支持跳转路径中的关键文档

### 用户价值

帮助用户理解知识网络中的桥梁节点，支持专题整合和结构优化。

## 6. 可操作建议 Actionable Suggestions

### 功能目标

将分析结果转化为可执行整理动作，而不是停留在数据展示。

### 核心能力

- 建议补充链接的孤立文档
- 建议升级为主题页的高热度文档
- 建议归档的长期沉没资料
- 建议重点维护的桥接文档
- 提供建议理由说明

### 用户价值

帮助用户把“看见问题”直接转化为“开始处理问题”。

## 九、关键交互流程

### 流程 1：发现核心文档

1. 用户打开脉络镜
2. 查看文档块引用热度排行
3. 选择高热度文档进入详情
4. 查看该文档连接的主题分布、传播路径与原始引用证据
5. 跳转原始文档完善为主题页或索引页

### 流程 2：发现知识簇

1. 用户进入主题社区发现模块
2. 查看已形成的主题社区列表
3. 选择一个社区查看内部高连接文档
4. 识别缺少统一主题页的社区
5. 跳转关键文档进行结构整理

### 流程 3：修复知识断裂

1. 用户进入孤立文档块识别模块
2. 查看孤立文档与长期沉没内容列表
3. 按更新时间排序，优先处理近期内容
4. 跳转原始文档补充引用、标签或归档处理

### 流程 4：观察主题变化

1. 用户进入时间趋势分析模块
2. 选择近 30 天窗口
3. 查看哪些主题活跃度上升，哪些主题持续沉没
4. 根据结果决定继续沉淀或做归档整理

## 十、数据模型建议

### 核心对象

- DocumentNode：文档块网络节点对象
- DocumentEdge：文档块之间的引用关系边对象
- CommunityCluster：主题社区对象
- TrendMetric：趋势指标对象
- SuggestionItem：整理建议对象
- ReferenceEvidence：原始块级引用证据对象

### 分析粒度定义

- 第一阶段默认以文档块作为最小分析单位，而不是普通内容块
- 网络中的节点是文档块，边是“文档块 A 引用文档块 B”的聚合关系
- 同一文档内多个块指向同一目标文档时，在网络层聚合为一条文档级关系，并保留引用次数作为权重
- 块级引用明细只作为解释证据层存在，用于回答“为什么这个文档被判定为高热度/强连接”

### 设计原则

- 结果必须可解释，不能只输出抽象分数
- 分析结果必须能回指原始文档，并在需要时展开原始块级证据
- 视图和分析逻辑解耦，便于后续扩展
- 保持与思源块模型一致，不建立独立平行知识图数据库
- 默认优先控制颗粒度，避免因块级噪声导致排行和社区结果失真

## 十一、MVP 范围

第一阶段建议只做：

- 文档块引用热度排行
- 主题社区发现
- 孤立文档块识别
- 基础时间趋势分析
- 关系传播路径的简化版
- 可操作建议列表
- 原始文档跳转能力
- 原始引用证据查看能力

这组功能已经可以覆盖“发现核心内容、发现知识簇、发现断裂内容、做整理决策”四类核心需求。

## 十二、暂不纳入第一阶段

- 大型交互图谱画布
- 3D 图谱渲染
- 高级网络中心性全量指标面板
- AI 自动生成完整知识结构报告
- 多人协作网络对比分析

## 十三、成功指标

### 使用指标

- 分析器打开频次
- 热度排行、社区分析、孤立文档模块点击率
- 从分析结果跳转原始文档的次数
- 整理建议被采纳的次数

### 结果指标

- 孤立文档数量下降趋势
- 高价值主题页建设数量提升
- 主题社区内部连接密度提升
- 长期沉没资料比例下降

### 体验指标

- 用户识别核心文档的时间缩短
- 用户对建议可解释性的满意度
- 分析结果复访率

## 十四、风险与难点

### 风险 1：做成“更复杂的图谱”

如果输出仍然以画图为主，用户可能觉得新鲜但不持续使用。

### 风险 2：分析结果不可解释

如果只给出热度分数或社区结果，但无法解释原因，用户很难信任结果。

### 风险 3：路径分析复杂度过高

关系传播路径计算容易变重，第一阶段必须控制范围与深度。

### 风险 4：社区发现结果语义弱

算法能聚成簇，不代表用户能立刻理解其主题含义，因此需要结合标签、属性或高频文档做辅助解释。

## 十五、后续路线图

### Phase 1

- 热度排行
- 社区发现
- 孤立文档块识别
- 时间趋势分析
- 简化路径分析
- 建议列表

### Phase 2

- 更丰富的筛选器
- 更强的趋势图表
- 社区语义标签建议
- 与知识库体检仪表盘联动

### Phase 3

- 与查询构建器联动
- 与自动化规则联动
- 高价值文档自动候选专题页
- 半自动结构修复建议

## 十六、结论

脉络镜的核心价值，不是展示知识网络，而是帮助用户以文档块为分析单位理解知识结构、识别关键节点、发现断裂区域，并把分析结果转化为具体整理动作。

一句话定义：

让图谱从“可视化摆设”变成“知识结构优化工具”。
